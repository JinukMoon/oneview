# 파일 뷰어 앱 — 설계 철학 & 기능 정리

> 이 문서는 앱의 **이름과 로고를 생성**하기 위한 입력용입니다.
> 앱의 컨셉, 철학, 기능, 분위기를 담았습니다. 맨 아래에 이름/로고 생성용 가이드가 있습니다.

---

## 1. 한 줄 소개

**휴대폰에서 어떤 문서든(특히 한글 HWP 포함) 광고 없이, 앱 하나로 바로 열어보는 무료 뷰어.**

---

## 2. 만든 이유 (문제의식)

- 휴대폰에서 PPT, PDF, 엑셀, 한글(HWP) 등을 열려면 **포맷마다 앱이 다 따로**다.
- 그 앱들은 대부분 **광고가 많고, 무겁고, 로그인을 요구**한다.
- 특히 **HWP/HWPX(한글 문서)는 휴대폰에서 멀쩡히 볼 무료 수단이 거의 없다.** (한국 사용자의 가장 큰 불편)
- 카톡·메일로 받은 파일을 누를 때마다 **"어떤 앱으로 열까" 고르는 행위 자체가 번거롭다.**

→ 그래서 **"하나의 진입점에서 다 열리는, 광고 없는 개인용 종합 뷰어"**를 만든다.

---

## 3. 설계 철학 (핵심 원칙)

1. **가볍고, 광고 없고, 공짜.** 군더더기 없이 "보는 것"에만 집중한다. 로그인·계정·과금 없음.
2. **하나의 진입점.** 카톡·메일·파일앱에서 파일을 누르면 곧바로 이 앱에서 열린다. 사용자는 앱을 고를 필요가 없다.
3. **직접 그리되, 못 그리는 건 똑똑하게 위임한다.** 대부분의 포맷을 앱 안에서 직접 렌더링한다. 하지만 오픈소스로 완벽 재현이 불가능한 것(예: PowerPoint 애니메이션 슬라이드쇼)은 고집부리지 않고, 그걸 가장 잘하는 앱(PowerPoint)으로 한 번에 넘긴다. **자존심보다 사용자 편의.**
4. **오프라인 · 프라이버시 우선.** 모든 처리는 휴대폰 안에서 이뤄진다. 파일을 서버로 보내지 않는다.
5. **한국 문서를 일급 시민으로.** HWP/HWPX를 핵심 기능으로 다룬다. 이게 다른 글로벌 뷰어들이 비워둔 빈자리이자 이 앱의 정체성이다.
6. **눈에 편하게.** 다크 모드 기본. 문서 가독성과 야간 사용을 배려한다.

---

## 4. 타겟 사용자

- 카톡·메일로 **온갖 형식의 문서를 자주 받는 한국 사용자** (직장인, 대학원생, 공무원, 연구자 등)
- 광고 범벅 뷰어에 지친 사람
- HWP를 휴대폰에서 제대로 못 봐서 답답했던 사람

---

## 5. 기능

### 앱 안에서 바로 보는 포맷
| 형식 | 방식 | 품질 |
|---|---|---|
| **PDF** (한글 포함) | 내장 렌더 + 한글 폰트 | 완벽 |
| **HWP** (.hwp, 한글) | 시각 렌더 | 원본 레이아웃 |
| **HWPX** (.hwpx, 신형 한글) | 내용 추출 | 글 내용 위주 |
| **Word** (.docx) | 내장 렌더 | 좋음 |
| **Excel** (.xlsx/.xls) | 표 렌더 | 좋음 |
| **PowerPoint** (.pptx) | 정적 슬라이드 미리보기 | 빠른 훑기용 |
| **이미지** (jpg/png/gif/webp 등) | 내장 | 완벽 |
| **텍스트** (txt/csv/md/json/xml 등) | 내장 (한글 인코딩 자동 감지) | 완벽 |

### 똑똑한 위임 (외부 앱으로 자동 전달)
- PPT의 **진짜 애니메이션 슬라이드쇼**가 필요하면 → 버튼 한 번으로 PowerPoint로 넘김
- 레거시·희귀 포맷(.doc/.ppt 구버전, .rtf, .odf, .epub 등) → 가장 알맞은 설치된 앱으로 자동 전달

### 사용성
- **헤더 고정 + 콘텐츠만 줌인/줌아웃** (핀치 + 버튼, 25%까지 축소 가능)
- **다크 모드** 기본
- **다크 리더(야간 반전)** — 흰 문서를 어둡게 반전 *(예정)*
- **최근 본 파일 목록** — 카톡 다시 안 뒤져도 재열람 *(예정)*
- **공유/다른 앱으로 보내기** *(예정)*
- 카톡·메일 "열기 목록"에 자동 등장 (파일 누르면 바로 연결)

---

## 6. 차별점 (한 문장씩)

- **HWP를 휴대폰에서 제대로 보는 거의 유일한 무료·무광고 수단.**
- 포맷마다 앱을 갈아타지 않아도 되는 **단 하나의 문 (one door).**
- 못하는 척하지 않고, 못하는 건 솔직히 더 잘하는 앱에 넘기는 **겸손한 똑똑함.**
- 서버 없이 **폰 안에서 끝나는** 프라이버시.

## 8. 분위기 & 성격 (이름·로고용 핵심)

**키워드**
`간결함` · `신뢰감` · `만능(다 열린다)` · `한국적(한글 문서)` · `가벼움` · `광고 없는 깨끗함` · `하나의 문/창` · `다크 모드`

**톤**
- 똑똑하지만 잘난 척하지 않는다. 조용하고 믿음직한 도구.
- 화려함보다 **실용·미니멀**.
- 약간의 위트는 OK, 유치하지 않게.

**비주얼 방향**
- 기본 테마: **다크 (배경 #0d1117 계열)**, 포인트 컬러 **파란색 (#2f81f7)**.
- 로고는 단순한 심볼이 좋음 — 예: 창/문, 겹친 문서, 렌즈/뷰파인더, 한글 자모 모티프, 폴더가 열리는 형상 등.
- 작은 런처 아이콘에서도 알아볼 수 있게 **단순하고 대비 강한** 형태.

**피하고 싶은 느낌**
- 광고 가득한 싸구려 유틸 앱 느낌
- 너무 기업적/딱딱한 오피스 느낌
- 과한 그라데이션·디테일 (아이콘이 작아지면 뭉개짐)

## 9. 이름/로고 생성 LLM에게 주는 지시 (가이드)

> 위 컨셉을 바탕으로:
>
> 1. **앱 이름 후보 8~12개**를 제안해줘.
>    - 한글 이름과 영문 이름을 섞어서. (한국 사용자 대상, 한글 문서가 핵심)
>    - "다 열린다 / 하나의 창 / 가볍다 / 한글까지 본다"는 정체성이 드러나면 좋음.
>    - 발음·기억 쉬움, 앱스토어/도메인 충돌 적을 법한 것 우선.
>    - 각 이름마다 한 줄 의미 설명 첨부.
>
> 2. 추천 이름 2~3개에 대해 **로고 컨셉**을 텍스트로 묘사해줘.
>    - 다크 배경 + 파란 포인트, 단순한 심볼, 작은 아이콘에서도 식별되는 형태.
>    - 가능하면 이미지 생성 프롬프트(영문)도 함께 제시.
